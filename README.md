# NIST CSF 2.0 Control Gap Assessment – Identity & Security Controls  

## Project Overview

This project simulates a real-world Control Gap Assessment using the NIST Cybersecurity Framework (CSF) 2.0.

The objective was not to list controls, but to evaluate how well an organization’s current practices align with control intent, identify governance and security gaps, assess risk impact, and recommend practical remediation actions.

This project reflects the kind of structured analysis performed in GRC, IT audit, compliance, and security governance roles.

## Project Goals

- Apply NIST CSF 2.0 to a practical organizational scenario

- Translate high-level controls into real operational conditions

- Identify control gaps related to identity, data, assets, and third-party risk

- Assess business and security impact of missing controls

- Recommend governance-focused remediation actions

- Assign control ownership and realistic timelines

# Framework Used

## NIST Cybersecurity Framework (CSF) 2.0

Function | Control Areas Assessed
---      | ----
Identify | Asset inventory, data inventory, external dependencies
Protect  | Identity & access governance, awareness & training

# Key Control Domains Covered
## Identity & Access Management (IAM)
- Identity lifecycle management (PR.AA-01)
- Identity proofing & credential binding (PR.AA-02)
## Asset Management
- Employee laptops and endpoint inventory (ID.AM-01)
- Ownership and lifecycle tracking of devices
## Data Governance
- Inventory and classification of customer PII (ID.AM-07)
- Metadata tracking (classification, ownership, storage location)
## Human Risk & Awareness
- Phishing and email threat awareness (PR.AT-01)
# Control Gap Assessment Structure
Element	| Purpose
----    | ----
Current State	| Describes existing condition factually
Gap Identified|	What is missing vs control intent
Risk Impact	| Business/security consequences if gap persists
Recommendation	| Targeted action to close the gap
Owner |	Role responsible for remediation
Target | Date	Realistic timeline for implementation
# Key Learning Outcomes
- Controls fail more from unclear ownership than missing tools
- Identity risks often stem from lifecycle governance, not authentication technology
- Organizations frequently know where systems are, but not where their data resides
- Vendor risk is a governance challenge, not just a technical one
- Security awareness is a people control, not an email system control
